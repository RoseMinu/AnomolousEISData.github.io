Detecting Anomalous Battery Cells

Author: Anomalous Cells Team
Last updated: 2024-06-05
Project mission Build a transparent workflow for flagging out-of-family battery cells using electrochemical impedance spectroscopy (EIS) data collected under rate capability testing. Each page in this guide walks through a reproducible analysis pipeline you can adapt for new cohorts of cells.

Large fleets of advanced batteries are only as reliable as the rarest cell in the pack. By combining statistical analysis with interpretable models, we can surface subtle electrical signatures that reveal latent manufacturing defects before they cascade into safety or warranty events. This microsite documents our reference implementation and the rationale behind key design choices.

What you'll find here

  • Data specification: File formats, metadata expectations, and recommended feature engineering for impedance spectra.
  • Exploratory analysis: Visual patterns that distinguish nominal and anomalous cells across state-of-charge sweeps.
  • Model library: Baseline anomaly detectors and guidance on tuning thresholds for production.
  • Reporting: Interpreting results, quantifying risk, and communicating findings with stakeholders.

Repository structure

The GitHub repository is organised so you can quickly plug in new datasets or extend the modelling stack. Use the navigation menu to deep dive into each stage, or skim the quick start checklist below to run the entire workflow end to end.

Folder Contents Notes
data/ Raw and processed impedance measurements, metadata manifests, and quality flags. Not versioned in Git; pull from secure storage before running notebooks.
notebooks/ Exploratory data analysis, feature creation, and model evaluation notebooks. Execute in order; outputs feed directly into the modelling scripts.
src/ Reusable Python modules for preprocessing, feature extraction, and anomaly scoring. Packaged to support both notebook experimentation and CLI workflows.

Quick start

  1. Clone the repository and create a dedicated Conda or virtual environment.
  2. Download the latest impedance dataset bundle to data/raw/ and run make features to generate training matrices.
  3. Open the notebooks in ascending order to validate data health and identify candidate features.
  4. Run python -m src.models.evaluate to train baseline detectors and export performance reports.
  5. Review the Results page for guidance on interpreting anomaly scores before deploying thresholds.
